# Personal OS - Design System Specification V1

## Design Philosophy

Theme:
Architectural Brutalism

Inspired by:

* Exposed concrete
* Raw plaster
* Editorial layouts
* Architecture portfolios
* Design journals
* Physical notebooks

The application should feel:

* Calm
* Structured
* Intentional
* Tactile
* Focused

Not:

* Futuristic
* Cyberpunk
* SaaS-looking
* Glassmorphic
* Over-animated
* Neon

The interface should disappear behind the work.

---

# Core Design Principles

1. Tasks before analytics
2. Content before decoration
3. Structure before color
4. Space before borders
5. Progress before metrics
6. Function before customization

# Design Rule

If a component exists only because it looks cool, remove it.

If a component helps complete a task faster, keep it.
