# Personal OS - Dashboard Specification V1

## Core Philosophy

The Dashboard is not an analytics page.

The Dashboard is a command center.

When opening the app, the first thing visible should be:

"What should I do next?"

Analytics come later.

---

# Dashboard Layout

1. Today's Queue
2. Current Focus
3. Quick Stats
4. Recent Activity
5. Analytics

---

# Section 1 - Today's Queue

This is the primary section.

It appears at the very top.

All actionable checklist items from across the system are collected here.

Sources:

* Skills
* Projects
* Goals
* Workouts
* Health Tasks

---

Status

---

# Queue Sorting Rules

Priority 1

Scheduled Tasks

Sorted by time ascending.

Example

09:00

10:30

18:00

20:30

---

Priority 2

Unscheduled Tasks

Sorted alphabetically.

Example

□ Build Dashboard

□ Learn Authentication

□ Learn FastAPI

□ Write Project Notes

---

Priority 3

Everything Else

Remaining checklist items.

---

# Scheduling

Any checklist item may optionally contain:

Scheduled Time

Example

□ Learn Playwright
Time:
09:00

---

Items without time remain unscheduled.

---

# Completion Behavior

When completed:

Item moves to:

Completed Today

at the bottom.

---

Example

Completed Today

✓ Learn Playwright Selectors

✓ Push Workout

✓ Log Weight
