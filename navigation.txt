# Personal OS - Navigation & Information Architecture Specification V1

## Purpose

The navigation system should make the application predictable and easy to understand.

A user should always know:

* Where they are
* What they can do next
* How to reach any section within 2–3 clicks

The structure should reflect the hierarchy of life management rather than database tables.

---

# Navigation Philosophy

Primary Question:

"What do I need to do today?"

Secondary Question:

"What am I working toward?"

Therefore:

Dashboard comes first.

Analytics comes later.

Settings come last.

# Navigation Rules

Never exceed 2 levels of nesting.

Avoid hidden pages.

Avoid deeply nested menus.

Avoid separate dashboards for each module.

Keep navigation predictable.

A user should be able to access any major feature within three clicks or fewer.

---

# Version 1 Scope

Must Include

Sidebar Navigation

Dashboard

Goals

Skills

Projects

Health

Fitness

Analytics

Settings

Global Search

Quick Actions

Breadcrumbs

---

Do Not Include

Multi-Workspace Navigation

Team Navigation

Project Spaces

Complex Nested Menus

Role-Based Navigation

Plugin Navigation

The navigation system should remain simple, task-oriented, and centered around the Dashboard as the user's command center.
